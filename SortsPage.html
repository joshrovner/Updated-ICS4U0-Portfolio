Tutorial Continued - Solving a Problem Using Searches and Sorts

Let's try to solve a problem: Given an integer array of length 2 - 10, use one method to sort the array from least to greatest. Then, return the first position of the integer "3" in the array. If there are no 3's, return -1.

To solve this problem, we must first sort the array. Since it will have a maximum of only 10 elements, we know that insertion sort will be an effective algorithm. Thus, we can implement it:

{

  { 
    //sets key to current position
    int key = arr[i]; 
    int j = i - 1; 
    
    //loops backwards through the array until the correct spot for the key is found
    while (j >= 0 && arr[j] > key) 
    { 
      arr[j + 1] = arr[j]; 
      j = j - 1; 
    } 
    
    //key is inserted in the correct position
    arr[j + 1] = key; 
  } 
}

public int findPos3 (int[] array)
{
  //insertion sort
  for (int i = 1; i < array.length; i++) 
  { 
    //sets key to current position
    int key = arr[i]; 
    int j = i - 1; 
    
    //loops backwards through the array until the correct spot is found
    while (j >= 0 && arr[j] > key) 
    { 
      arr[j + 1] = arr[j]; 
      j = j - 1; 
    } 
    
    //key is inserted in the correct position
    arr[j + 1] = key; 
  } 
  
  //sequential search
  for (int i = 0; i < array.length; i++)
  {
    //if a 3 is found, the index will be returned
    if (arr[i] == 3)
      return i;
  }
  //-1 is returned if no 3 was found
  return -1;
}

Lines 21 through 29 loop through the now-sorted array, and check each element for a 3. If one is found, the method immediately returns the index at which it was found and exits. If the for loop checks the whole array and doesn't find any threes, it returns -1.

Search and Sort Tips

Avoid IndexOutOfBoundsExceptions by tracing your algorithm, instead of immediately trying to alter your code.
Use the most efficient sort for each scenario to save time and space.
If a search or sort doesn't seem to work, print out every element in the list before and after the algorithm to see what, if anything, was changed.
