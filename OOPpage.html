public class Rectangle
{
  private int length;
  private int width;
  static int numRectanglesMade = 0;
  
  public Rectangle (int length, int width)
  {
    this.length = length;
    this.width = width;
    numRectanglesMade++;
  }

  public int getLength ()
  {
    return length;
  }

  public void setLength (int newLength)
  {
    if (newLength > 0)
      length = newLength;
  }

  public int getWidth ()
  {
    return width;
  }

  public void setWidth (int newWidth)
  {
    if (newWidth > 0)
      width = newWidth;
  }

  public String toString () 
  {
    return "This is a rectangle with length " + length + " and width " + width + ".";
  }

  public int area ()
  {
    return length * width;
  }

  public int perimeter ()
  {
    return 2 * (length + width);
  }
  
  public double getDiagonal ()
  {
    return Math.sqrt (Math.pow (length, 2) + Math.pow (width, 2));
  }
}

As we can see, the class contains a constructor with two parameters: length and width. This means that whenever an object of the class is created, the length and width will be specified. Also in the constructor is the keyword "this", which implies that class variables are being referred to, not the parameters. So, the statement this.length = length; means that the class variable "length" is being given the value of the parameter "length". The last line increments the static variable numRectanglesMade - remember that static variables apply to the class, not the object, so this variable simply stores the number of objects that have been made of the class.

Also in the class are two get methods and two set methods. Get methods, or accessor methods, return values of certain variables. Nothing else is done inside of them - in the code, all they do is return the value of the according variable. Set methods, or mutator methods, take parameters to alter variables. These are not limited to one line - processing can be done in mutator methods.

The toString method provides a description of the Rectangle object in an understandable sentence.

Now, you may be wondering, why are the variables private and the methods public? This is the theory of encapsulation, where variables (fields) should not be accessed by another class. Instead, their values can be found and altered through accessors and mutators.

public class Square extends Rectangle
{
  public Square (int sideLength)
  {
    super (sideLength, sideLength);
  }

  public double getDiagonal ()
  {
    return getLength () * Math.sqrt (2);
  }
  
  public double getVolumeOfCorrespondingCube ()
  {
    return Math.pow (getLength (), 3);
  }
  
  public String toString ()
  {
    return "This is a square with side length " + getLength () + ".";
  }
}

In the class header, we see the phrase "extends Rectangle" at the end. This means that Square can be considered a subclass of Rectangle, which is the superclass. This demonstrates the concept of inheritance - subclasses inherit all public or protected fields and methods of their superclass. So, the Square class will be able to use the methods of Rectangle, as they are all public, but will not be able to access the variables, which are private.

Moving on, the constructor takes one parameter: the side length. Then, it uses the super () method, which invokes the constructor of Rectangle to create a new object with the same length and width.

Dealing with the class methods, getDiagonal () and toString () are "overridden", as they have the same names and parameter lists as methods in Rectangle, but provide different implementations. Finally, getVolumeOfCorrespondingCube () is a method unique to Square, so it cannot be accessed by any Rectangle objects.

public class RectangleDriver
{
  public static void main (String[] args)
  {
    Rectangle r = new Rectangle (5, 6);
    System.out.println (r.toString ());
    System.out.println ("Area = " + r.area ());
    System.out.println ("Diagonal = " + r.getDiagonal ());
    System.out.println  (r.numRectanglesMade);
    System.out.println (Rectangle.numRectanglesMade);
    
    System.out.println ();
    
    Square s = new Square ();
    Square s = new Square (5);
    System.out.println (s.toString ());
    System.out.println ("Perimeter = " + s.perimeter ());
    System.out.println ("Diagonal = " + s.getDiagonal());
    System.out.println ("Volume of Corresponding Cube = " + s.getVolumeOfCorrespondingCube ());
    
    System.out.println ();
    
    Rectangle r2 = new Square (6);
    System.out.println (r2.toString ());
    System.out.println (r2.getVolumeOfCorrespondingCube ());
    
    Square s2 = new Rectangle (4, 4);
  }
}

This class only contains a main method, as it is used simply for executing tasks. First, we create a Rectangle object using the keyword "new". Whenever this is used, it automatically means an object is being created. In this line, we also pass two integers to the constructor - 5 and 6. Looking back at the Rectangle class constructor, we remember it required a length and width parameter, so 5 will be set to the length, and 6 will be set to the width. Thus, we can now imagine that we have a real 5 x 6 rectangle. Next, we call the toString () method, which prints out the rectangle's dimensions. Then, we call and print out the area and diagonal methods, which both return their respective values. After, we print out the value of numRectanglesMade on the current Rectangle object. However, recall that static variables do not apply to each individual object of a class; they only apply to the class itself. Thus, line 10 correctly accesses the value of the variable.

Next, we create a new Square object with no parameters. This causes an error, as the Square class constructor requires a sideLength parameter. So, in the next line, we create a new object with 5 as the side length. Then, we call the toString () method again, but interestingly, this time it prints out information about being a square. This is because we utilised the overridden toString () method in the Square class. Then, we call the perimeter () method on the Square. But remember, the Square class never had this method. How did it manage to use it? As stated earlier, subclasses can use non-private methods of their superclass. After this, we print out some more information about the Square.

Now, we move on to the third object, where we create a Square object of type Rectangle. We can do this because a Square is-a Rectangle, so it can fill in for a Rectangle - this is called "upcasting" as the Square is now considered as a Rectangle. Let's take a moment to think - after this declaration, which toString () method will be called? The correct answer would be the Square one, as the program initially finds the Rectangle one but then sees it is overriden by the Square class and uses that one instead. This demonstrates the concept of polymorphism - this object has the ability to take on many forms, calling methods in both Square and Rectangle. Next, we call the getVolumeOfCorrespondingCube method on this object. But, an error is caused because this method could not be found in the Rectangle class. If it were in both classes like toString (), it would have called the Square method, but since the object is considered a Rectangle and this method is not present in the Rectangle class, it will not be found. Finally, we create an Rectangle object of type Square. This will cause an error, as a Rectangle is-not-a Square, and cannot fill a Square position.

If we changed the class header of the Rectangle class from public class Rectangle to public abstract class Rectangle and added the header public abstract int getVolumeOfAccordingCube (); into line 11, we would be dealing with an abstract class. The Rectangle object could no longer be made, but the Square class could still be upcast to type Rectangle, and Rectangle methods could be used. If we wanted to use the Shape interface's methods for the Rectangle class, we would have to change the class header public class Rectangle to public class Rectangle implements Shape. Recall that a class implementing an interface must provide implementations for all of its methods, so the two get and set methods in Shape would have implementations in Rectangle.

OOP Tips

Ensure to provide implementations for all abstract methods.
Keep variables private and provide accessors and mutators to access them.
Use the is-a check to see whether certain types can be substituted for others.
